Linux 與嘸蝦米(feat. Fcitx)

發表於 2025-12-21 分類於 boshiamy
文章字數： 2.8k 所需閱讀時間 ≈ 5 分鐘

前言

Fcitx，中文名稱是「小企鵝輸入法」。

本篇紀錄，主要是把各大發行版分類，再分別整理出安裝 Fcitx 與嘸蝦米輸入法時的過程紀錄與建議。

希望這樣的紀錄，對有需要的伙伴而言，能夠有一定的參考度。

各發行版之安裝建議(依英文字母排序)

Arch Linux 及其衍生發行版

可以優先考慮 Fcitx5。

  1. 安裝 Arch Linux 與嘸蝦米
  2. Manjaro XFCE 23.0 與嘸蝦米
  3. 來玩 EndeavourOS Apollo 22.1
  4. 來玩 Garuda Linux (KDE Dr460nized Edition)
  5. ArcoLinuxL 與嘸蝦米
  6. CachyOS

Debian 與 ubuntu 及其衍生發行版

Fcitx5(fcitx5-table-boshiamy)

  1. 嘸蝦米的獨立套件：原本，「嘸蝦米」是打包在 fcitx5-table-extra 這支套件裡，然而，現在已經有單獨打包「嘸蝦米」成一支套件問世：fcitx5-table-boshiamy。截至目前，我觀察到部分 Debian 與 ubuntu 及其衍生發行版開始將 fcitx5-table-boshiamy 收錄至軟體庫，讓選用 Fcitx5，並且只想單純地服用「嘸蝦米」的伙伴，可以選擇安裝 fcitx5-table-boshiamy 這支套件即可。
  2. 視窗介面安裝：在 Linux Mint 22.1 上，可以使用「軟體管理員」介面完成輸入法框架及輸入法表格相關套件的安裝，並且後續的設定上，也都是在視窗介面完成。在 Linux Mint 22.1 之前，能夠在視窗介面就完成安裝的發行版，我目前只想得到 openSUSE。

基於上述 2 點，我先把 fcitx5-table-boshiamy 的安裝方式獨立出來。

  1. [無需指令] 在 Linux Mint 22.1 Cinnamon 上安裝嘸蝦米輸入法
  2. Zorin OS 18 與嘸蝦米

Fcitx5(fcitx5-table-extra 軟體庫版)

「ubuntu 24.04 LTS」及其衍生發行版，已經將 fcitx5-table-extra 收納至軟體庫，所以，先是一道指令將套件安裝進來，接著可以從圖形介面新增「嘸蝦米」就能搞定。

  1. ubuntu 24.04 LTS 與嘸蝦米
  2. Linux Mint 22 與嘸蝦米 (feat. fcitx5-table-extra)

Fcitx5(fcitx5-table-extra 自行處理與服用版)

fcitx5-table-extra 需要自行處理與服用，本篇貼文運用的方法是跑一趟 pkgs.org，下載相關檔案後再做一些處理，同樣可以服用，只是過程比較麻煩一些。

  1. Debian 12 與嘸蝦米
  2. 在 ubuntu 22.04.1 LTS 上用 Fcitx5 搭配嘸蝦米 (feat. fcitx5-table-extra)
  3. Linux Mint 21.1 與嘸蝦米 (feat. fcitx5-table-extra)
  4. 來玩 LMDE 6

fcitx(4)

以下的發行版，選用 fcitx(4) 是比較方便的決定，因為，只要下完 sudo apt install fcitx-table-boshiamy 這道指令就幾乎搞定一半。

  1. ubuntu 23.04 與嘸蝦米
  2. 在 ubuntu 22.04 LTS 上輕鬆安裝嘸蝦米
  3. 來玩 MX Linux MX-23
  4. 來玩 elementory OS 7.1
  5. Zorin OS 17.3 Core 與嘸蝦米
  6. 來玩 LMDE 5

Fedora 及其衍生發行版

Fcitx5 是首選，在 Terminal 下 sudo yum install fcitx5* 或 sudo dnf install fcitx5* 指令來服用。

  1. 在 Fedora 41 上輕鬆安裝嘸蝦米(feat. Fcitx5)
  2. 來玩 risiOS 37.1.1

openSUSE

建議使用 Fcitx5，有可能未來也只有 Fcitx5 可選。

使用 YaST 圖形介面模式進行相關套件安裝。

  1. openSUSE Leap 15.5 與嘸蝦米 (feat. Fcitx5)
  2. openSUSE Tumbleweed Snapshot20231018 與嘸蝦米 (feat. Fcitx5)

使用 Myrlyn 圖形介面模式進行相關套件安裝。

  • openSUSE Leap 16.0 Beta

Slackware

原則上，fcitx(4) 與 fcitx-table-boshiamy 都不用再另外安裝，使用者要做的是一點點設定。

  • 來玩 Slackware 15.0

後記

  1. Linux 上的嘸蝦米輸入法走過不少輸入法平台，其中 SCIM 與 gcin 似乎已然是過去式。近幾年，比較為人所用的主要就是 iBus 與 Fcitx。選用 iBus 的話，拿行易有限公司官網的表格檔服用，肯定是最佳的選擇。

  2. 談到 Fcitx 的話，我個人會用 fcitx(4) 與 Fcitx5 來區分這 2 代的輸入法框架，版本間的選用，我認為，可以考量 2 個重點。
    (1) 軟體倉庫的收納程度：隨著每支發行版的迭代更新，越來越多的系統預載 Fcitx5，Fcitx5 的相關套件收納程度也越來越完整。除了少數幾支發行版，還是以安裝與使用 fcitx(4) 較為方便之外，相信，套件收錄已經日趨完整的 Fcitx5，將會是中文輸入法框架主要的選擇。
    (2) 系統環境支援的考量：Wayland 的趨勢使然，也會讓使用者選擇 Fcitx5。
