Windows 非預期重新啟動

發表於 2024-12-28 分類於 windows
文章字數： 1.2k 所需閱讀時間 ≈ 2 分鐘

前言

目前，我所使用的 Windows 桌機是 Acer Aspire TC-1660。最近一直遇到系統在非預期的狀況下，自己就重新啟動的狀況，而本文主要是紀錄我的故障排除過程。

故障排除過程與想法

問題發生的時間點

  1. 這部機器出廠預載的系統是 Windows 10，後升級到 Windows 11。

  2. 使用一段時間的 Windows 11 後，我以重灌的方式，讓系統回到 Windows 10。

  3. 也就是從這個時間點開始，常常在我使用瀏覽器看影片（包括：Netflix、Youtube、Max 或 Disney+ 等）時，系統會卡住，接著進入黑暗畫面，然後，就是系統自己給自己重新啟動。

把問題先瞄準到顯示卡

  1. 一開始，我把問題鎖定在顯示卡，上網搜尋之後，也確實有不少伙伴有類似的情形。

  2. 這個時間點的排除方式，就是把顯示卡的驅動程式升好升滿。

  3. 至於，有沒有效果，答案是「有的」。只是，並沒有完全藥到病除。充其量，只能說，重新啟動的頻率變少，但實際上還是會發生。

嘗試透過更新 BIOS、Firmware 與 Windows 來解決？

  1. 我親自跑一趟官網，把最新的 BIOS 與 Firmware 一個接著一個都升級上去。

  2. 再把重灌並使用一年多的 Windows 10 升級到 Windows 11。

  3. 經歷過這些升級，無預期重啟的狀況，可以說完全感受不到有任何改善。

考慮更換電源供應器

  1. 會想到要更換電源供應器，主要是因為這部電腦，出廠的 CPU 風扇實在不優，買來不久，我就把風扇換掉。（詳可見：開箱 Acer Aspire TC-1660 Desktop feat. Cooler Master）

  2. 但，仔細回想一下，之前機器也是頂著新風扇運行好些日子，並沒有發生自己玩自己重新開機的狀況發生。

  3. 更何況，更換電源供應器是要花錢的，我認為除非非常必要，不然，我實在不太想先走這個解法。

  4. 不過，我得聲明的是…，我確實有點懶得去計算電源供應器能否負載目前機器上消耗的瓦數，如果算一算真的不太夠，或是太剛好的話，該花的錢，還是得花。

透過系統設定排除故障

我認為這是在不花錢的前提下，現階段我還能夠運用的方法。姑且，就把設定的過程，紀錄如下。

  1. 進入系統的「設定」介面。

  2. 點一下左邊的「設定」。

  3. 找到「系統進階設定」，並且點一下進入設定畫面。

  4. 在「進階」這個頁籤，找到「啟動及修復」欄位內的【設定】按鈕，並且點按進入。

  5. 在「系統失敗」欄位內，取消預設已勾選的「自動重新啟動」。

後記

  1. 設定完，可以明顯感受到，當我在瀏覽器看影片時，會有鼠標卡頓的狀況發生，但，還不至於讓系統重新開機。

  2. 但，我在使用 Oracle VirtualBox 上的 Fedora 41，還是遭遇到系統強制被關機的狀況。應該是我把「自動重新啟動」取消掉，所造成的結果。

  3. 由於我使用 Oracle VirtualBox 的頻率並不算高，所以，我決定再觀察一下，這樣的設定，所帶來的效果。
